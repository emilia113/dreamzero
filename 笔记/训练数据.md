# DreamZero 训练数据笔记

## 1. 基础格式：LeRobot v2

数据集需要遵循 LeRobot v2 格式：

```
your_dataset/
├── data/
│   └── chunk-000/
│       ├── episode_000000.parquet    # 轨迹数据
│       └── ...
├── videos/
│   └── chunk-000/
│       ├── observation.images.cam0/
│       │   ├── episode_000000.mp4    # 视频文件
│       │   └── ...
│       └── observation.images.cam1/
│           └── ...
└── meta/
    └── info.json                     # 必须包含: features, total_episodes, fps
```

---

## 2. DreamZero 额外要求（GEAR 格式转换）

在 LeRobot v2 基础上，DreamZero 需要运行转换脚本生成额外元数据：

```bash
python scripts/data/convert_lerobot_to_gear.py \
    --dataset-path /path/to/your_dataset \
    --embodiment-tag <EMBODIMENT> \
    --state-keys '{"joint_pos": [0, 6], "gripper_pos": [6, 7]}' \
    --action-keys '{"joint_pos": [0, 6], "gripper_pos": [6, 7]}' \
    --relative-action-keys joint_pos gripper_pos \
    --task-key annotation.task
```

---

## 3. 转换后生成的额外元数据文件

| 文件 | 内容 |
|---|---|
| `meta/modality.json` | state/action/video/annotation 的 key 映射和索引范围 |
| `meta/embodiment.json` | `{"embodiment_tag": "<EMBODIMENT>"}` |
| `meta/stats.json` | 每个特征的统计信息 (mean, std, min, max, q01, q99) |
| `meta/relative_stats_dreamzero.json` | 相对动作统计（action - reference state） |
| `meta/tasks.jsonl` | 唯一任务描述列表 |
| `meta/episodes.jsonl` | 每个 episode 的元数据（index, tasks, length） |
| `meta/initial_actions.npz` | 初始动作偏移量 |

---

## 4. Parquet 数据列要求

每个 episode 的 parquet 文件需要包含：

| 列名 | 类型 | 说明 |
|---|---|---|
| `observation.state` | float array | 机器人状态向量 (如 joint_pos + gripper_pos) |
| `action` | float array | 动作向量 (与 state 对应) |
| `observation.images.cam0` | 视频引用 | 摄像头视频 |
| `annotation.task` | string | 自然语言任务描述 |
| `timestamp` | float | 时间戳 |
| `episode_index` | int | Episode 索引 |
| `frame_index` | int | 帧索引 |

---

## 5. 数据归一化

支持多种归一化模式（在 YAML transform 中配置）：

| 模式 | 说明 |
|---|---|
| `q99` | 分位数归一化到 [-1, 1]（推荐） |
| `mean_std` | Z-score 标准化 |
| `min_max` | 最小最大归一化到 [-1, 1] |
| `scale` | 绝对最大值缩放 |
| `binary` | 二值化阈值 |

---

## 6. 相对动作 (Relative Action)

DreamZero 默认使用**相对动作**（当前动作相对于当前状态的差值），而非绝对动作：
- `relative_action: true`
- `relative_action_keys` 需要列出同时存在于 state 和 action 中的 sub-key 名称
- 需要 `relative_stats_dreamzero.json` 提供相对动作的统计信息

---

## 7. 视频要求

- 支持多摄像头输入（通常 3 个：外部左/右 + 腕部）
- 分辨率：训练时 resize 到 320x176 或 480x256
- 帧数：通常采样 25 帧（delta_indices 0-24），训练时设 num_frames=33
- 视频解码后端：`decord`
